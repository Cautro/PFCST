# PFCST — мобильное приложение + Express backend

Этот репозиторий содержит:
- `App/` — мобильное приложение на React Native (Expo).
- `Backend/` — сервер на Express с JSON базой данных (`db.json`).

Ниже — подробный порядок запуска, параметры и частые вопросы.

---

## 1. Требования

Установите:
- Node.js 18+ (рекомендуется LTS).
- npm (идет вместе с Node.js).
- Git (необязательно, но удобно).
- Expo Go на телефоне (если будете запускать на реальном устройстве).
- Android Studio / Xcode (если нужен эмулятор).

Проверьте версии:
```bash
node -v
npm -v
```

## 3. Запуск backend

1) Откройте терминал и перейдите в папку backend:
```bash
cd Backend
```

2) Установите зависимости:
```bash
npm install
```

3) Запустите сервер:
```bash
npm run start
```

По умолчанию сервер стартует на порту `3001`.

Проверка:
```bash
curl http://localhost:3001/health
```
Должен вернуться JSON вида `{ "ok": true }`.

### Файл базы данных

Данные хранятся в `Backend/db.json`. Примерные пользователи уже добавлены:
- `admin / admin` — руководитель (полный доступ)
- `worker / worker` — сотрудник сварки
- `paint / paint` — сотрудник покраски

## 4. Запуск приложения (Expo)

1) Откройте новый терминал и перейдите в папку приложения:
```bash
cd App
```

2) Установите зависимости:
```bash
npm install
```

3) Запустите Expo:
```bash
npm run start
```

Откроется Expo DevTools и появится QR‑код.

### Запуск на телефоне (Expo Go)

1) Установите Expo Go.
2) Убедитесь, что телефон и ПК в одной сети Wi‑Fi.
3) Откройте Expo Go и отсканируйте QR‑код из DevTools.

### Запуск на Android Emulator

1) Запустите эмулятор через Android Studio.
2) В DevTools нажмите `Run on Android`.

## 5. Настройка доступа к backend

В `App/lib/api.ts` используется базовый URL:
- Android эмулятор: `http://10.0.2.2:3001`
- iOS/веб/локально: `http://localhost:3001`

Если запускаете приложение на реальном телефоне, замените `localhost` на IP‑адрес вашего ПК в локальной сети:
```ts
const BASE_URL = 'http://192.168.X.X:3001';
```
Проверьте, что firewall не блокирует порт `3001`.

## 6. Базовые сценарии работы

### Авторизация
Откройте экран `Вход/Авторизация` и используйте тестовые доступы:
- `admin/admin` — полный доступ ко всем вкладкам.
- `worker/worker` — доступ к `Заказы в процессе`, `ФИО пользователя`.
- `paint/paint` — доступ к `Покрас изделий`, `ФИО пользователя`.

### Смена и таймер
На экране меню доступна смена:
- `Начало смены` — запускает таймер.
- `Пауза` — ручная пауза.
- `Конец смены` — сохраняет смену в базу.

Таймер автоматически ставится на паузу:
- с 13:00 до 14:00 (обед)
- с 19:00 до 08:00 (ночное время)

### Роли доступа
Доступ к вкладкам:
- Руководитель / Зам / Техподдержка: `Заказы`, `Пользователи`, `Выполненные`, `Ассортимент`.
- Сотрудники: `Заказы в процессе`, `Покрас изделий` (только цех покраски), `ФИО пользователя`.

---

## 7. Основные экраны

- **Меню** — список доступных разделов и смена.
- **Заказы** — создание, редактирование, удаление, поиск.
- **Заказы в процессе** — производство изделий с кнопками `Начать/Пауза/Готово`.
- **Покрас изделий** — этап покраски после завершения изготовления.
- **Ассортимент** — список изделий и материалы.
- **Пользователи** — управление сотрудниками.
- **ФИО пользователя** — часы и продуктивность.

## 8. Частые проблемы

### 1) Приложение не видит сервер
Проверьте:
- backend запущен на `3001`
- правильный IP в `App/lib/api.ts`
- телефон и ПК в одной сети

### 2) Expo не видит телефон
Проверьте:
- одинаковый Wi‑Fi
- нет VPN

### 3) Ошибка на Android Emulator
На Android Emulator используйте `10.0.2.2` вместо `localhost`.
